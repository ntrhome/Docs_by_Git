====== Types ======
Created вівторок 02 липень 2019

Каждый компонент относится к одному из 4-х типов, имеющих свои API-интерфейсы для взаимодействия с системой и пользователем. Каждый тип имеет свой жизненный цикл - способ создания и завершения:
	- **Activities** (активность, операция, действие) - один экран с пользовательским интерфейсом.
	- **Services** (сервис, служба) - компонент без пользовательского интерфейса, работает в фоновом режиме и выполняет длительные операции, часто связанные с работой удаленных процессов.
	- **Content providers** (поставщик контента) - компонент без пользовательского интерфейса, путем транзакций управляет хранением данных приложения (в файловой системе, базе данных SQLite, Интернете, пр.), предоставляет данные по запросу других приложений (чтение, изменение). Например поставщик контактов.
	- **Broadcast receivers** (приемник вещания) - компонент практически без пользовательского интерфейса (может создавать в строке состояния уведомления для пользователя), реагирует на сообщения распространяемые в системе (в большей степени самой системой - экран выключился, аккумулятор разряжен, был сделан фотоснимок; или приложением - данные загружены и готовы к использованию), инициирует выполнение определенных действий при возникновении события, сообщение предоставляется как объект Intent.
