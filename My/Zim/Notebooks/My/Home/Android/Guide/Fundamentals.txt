====== Fundamentals ======
Created вівторок 02 липень 2019

https://developer.android.com/guide/components/fundamentals.html

**Песочница, права:**
Каждое приложение Android, установленное на устройстве, работает в собственной "песочнице" (изолированной программной среде):
	- Android - многопользовательская система Linux, в которой каждое приложение является отдельным пользователем;
	- по умолчанию система назначает каждому приложению уникальный ID пользователя Linux (этот идентификатор используется системой и неизвестен приложению); система устанавливает полномочия файлов приложения чтобы доступ к ним был разрешен только пользователю с его ID;
	- каждый процесс выполняется в собственной виртуальной машине (ВМ), так что каждое приложение выполняется изолированно от других;
	- по умолчанию каждое приложение выполняется в собственном процессе Linux; Android запускает процесс для выполнения приложения (его компонента), и завершает процесс когда он больше не нужен либо когда требуется освободить память.
Android реализует принцип предоставления минимальных прав, т.е. приложение имеет по умолчанию доступ только к тем компонентам, которые были ему разрешены. Приложение может несколькими способами как получить доступ к ресурсам системы так и предоставить доступ к своему ресурсу:
	- приложение может запросить разрешение на доступ к данным устройства, например к контактам пользователя, SMS-сообщениям, подключаемой карте памяти (SD-карте), камере, Bluetooth и др. Все разрешения должны предоставляться приложению при его установке;
	- двум приложениям можно назначить один ID. В этом случае они могут обращаться к файлам друг друга, и для экономии ресурсов можно выполнять приложения с одним ID в одном процессе Linux и использовали одну ВМ (приложения также должны быть подписаны одним сертификатом).
