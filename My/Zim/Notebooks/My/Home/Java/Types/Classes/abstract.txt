**Абстрактные методы и классы**

Иногда требуется определить такой класс для наследования, в котором объявлена только структура необходимой абстракции, обобщенная форма сущности для совместного использования подклассами, в каждом из которых могут быть реализованы намеченные методы и добавлены другие подробности. Такая ситуация характерна для случаев, когда абстракция сущностей сходна (например - геометрические фигуры), а реализация методов принципиально различна (расчет их площади).

С целью гарантирования переопределения намеченного метода существует модификатор **abstract**, указываемый перед абстрактным методом, а также перед классом, который содержит хоть один такой метод:
		abstract class B { ...
				abstract int length(String s); // метод задается без блока тела, с ";" в конце.
				... }

Абстрактный класс не может порождать объектов (new ...), также не существует абстрактный конструктор или абстрактный статический метод. В то же время, ссылка типа суперкласс возможна и вполне дееспособна в понятии восходящих ссылок, просто она должна указывать на объект полноценного субкласса (см. [[Classes:extends]] - Восходящие ссылки). 

Субкласс, производный от абстрактного, должен либо реализовать все абстрактные методы суперкласса, либо сам быть абстрактным.
