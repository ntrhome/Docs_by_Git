====== Methods ======
Created Среда 26 июня 2019

= = = = = = = = = = = = = = = =
**Методы:**
Методы могут возвращать результатом примитивные, массивы примитивных и объекты.

class ten
{
	int x;
	ten(int i){ x = i; }
	ten plus10(){ return new ten(x+10); }
} 
= = = = = = = = = = = = = = = =
**Параметр: переменная, принимающая переданное аргументом значение.**
Параметры входят в область видимости метода и работают аналогично объявленным в нем переменным.
= = = = = = = = = = = = = = = =

**Аргумент: передаваемое в метод значение или переменная.**
(Моё понимание. Кроме того, "ссылка" здесь понятие не С-шное, т.к. это не адрес в памяти, а скорее некий дескриптор на клетку таблицы, а указатель - другая такая клетка в таблице, содержащая дескриптор).

Аргумент в джавке передается всегда по значению, но означает это разное, в зависимости от того, ЧТО он передает:
- **АРГУМЕНТ-ПРИМИТИВ ПЕРЕДАЕТСЯ ПО ЗНАЧЕНИЮ**
		в стек копируется содержимое памяти по ссылке, ассоциированной с именнем, а содержится там значение примитива, т.о. примитив действительно передается по значению:
				int a = 5; 
				f(a); // будет передано 5
- **АРГУМЕНТ-ОБЪЕКТ ПЕРЕДАЕТСЯ ПО ССЫЛКЕ**
		в стек копируется содержимое памяти по ссылке, ассоциированной с именнем, а содержится там ссылка на объект, т.о. объект передается по ссылке:
				class B{...}
				B b; // по ссылке "b" объявили типа джавкин указатель на объект класса "B", которому пока что присвоен null
